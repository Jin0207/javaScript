Jquery의 필요성
	-DOM 객체 조작을 용이하게 한다.
		대상이 되는 노드를 쉽게 선택하고 동적으로 노드를 생성,수정,삭제하기가 용이하다.
	
	-CSS설정을 용이하게 한다.
	-Ajax통신을 쉽게 할 수 있다.
	-다양한 플러그인을 이용하여 손쉽게 시각적으로 미려하게 꾸밀 수 있다.
		(플러그인 : Jquery를 기반으로 만들어진 또 다른 라이브러리들)

----------------------------------------------------------------------------

Why? DOM객체 조작이 필요한가
==> 현재 JavaScript의 목적은 사용자의 요청에 따라 문서의 일부분만 바꿔주는 역할
	문서의 일부분만 바꿔 줄려면
	1. 대상이 되는 노드(태그) 찾아야됨
	2. 동적으로 노드를 생성하기도 해야함
	3. 동적으로 노드의 값을 수정하기도 해야함.
	4. 동적으로 노드를 삭제하기도 함



Jquery를 사용하는 주된 이유
	DOM 객체 조작을 하려고 사용하므로
	우리가 작성하는 JQuery 코드는 문서 객체가 모두 만들어지고 난 다음 동작해야한다.
	
	그래서 자바스크립트로 보면
	window.onload = function(){}
	이 이벤트 안에 코드를 작성해야한다.
	
	위의 이벤트를 JQuery에서는 다음과 같이 표현
	$(document).ready(function (){
	
	});
	
	jquery에서 추가된 함수를 사용하려면 $기호를 붙여야한다.
	
	위의 문자를 다음과 같이 간결하게 표현할 수 있다.
	$(function(){
	
	});
	
	
	<style> css를 설정할 수도 있고
	자바스크립트의 jquery로 css를 설정할 수 있는데
	어떨때 Jquery를 이용할까
		-- 동적으로 변화를 주고 싶을 때
		-- 프로그램에서 사용자의 요구에 따라 css를 바꿔야 할 때
	
	
	함수의 매개변수
	1개: 읽어오는 것
	2개: 설정하는 것
	
	--
	jquery가 제공하는 함수를  사용하는 기본 형식
	
	$(선택자).함수명( [매개변수(들)] );
	
	- 선택자: 태그이름, clss이름, id이름
	
	css함수
		스타일을 설정하거나 스타일 값을 읽어오는 함수
		
	css('속성명', 값') 
	==> css 설정(속성의 값을 설정)
	
	css('속성명')
	==> 속성의 값 읽어옴
	==> 맨 첫번째 것만 읽어온다.
	ex) h1태그마다 style color가 다를 때, color의 속성값을 읽어온다면 맨첫번째 태그의 속성값만을 읽어온다.
	
	---
	
	$("선택자").css("속성명", "값")
		선택자의 결과가 배열일 때, 모두 설정
	
	$("선택자").css("속성명")
		선택자의 결과가 배열이라도 맨 첫번째 요소의 값만 일거옹ㅁ
		
	--
	
	jquery 반복을 위한 함수 
	
	$.each(배열, 함수);
		==> 함수는 배열의 요소만큼 동작

	$.each(배열이름, function( index, item ){ });
		==> item은 배열의 요소를 하나씩 담아오는 변수
		==> index는 배열의 인덱스	
		
input태그의 값을 설정하고 읽어오기
	$("선택자").val(값); ==> 값 설정하기
	$("선택자").val(); ==> 값 읽어오기
	

**타이머 설정하기
	
	setTimeout(함수, 시간);
		==> 지금부터 정해진 시간에 도달하면, 함수를 1번 실행
	setInterval(함수, 시간);
		==> 정해진 시간마다 함수를 반복 동작
		
	함수를 미리 만들어 두고 함수이름을 써도 되고
	이름없는 함수를 바로 작성해도된다.
	
	시간은 밀리초로 설정해야 한다.
	(1000 = 1초)
	
	**노드의 삭제
		remove()	자기자신을 삭제
		empty()		자식 및 후손 노드들을 모두 삭제
